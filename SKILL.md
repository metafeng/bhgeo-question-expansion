# GEO 用户问题拓展

从公司/产品资料出发，按 L1→L2→L3→L4 层级拓展关键词和问题，并完成转化评分、Top 20 筛选和内容布局建议。一次输入，完整输出。

**层级定义：**
- **L1 关键词**：从资料中提炼的核心品类词/产品词
- **L2 拓展词**：L1 + 形容词修饰，覆盖不同属性和场景（用于 GEO 监测）
- **L3 核心问题**：L2 + 前后缀，形成标准化搜索问题（用于 GEO 收录监测）
- **L4 真实用户问法**：L3 的自然语言改写，模拟用户真实向 AI 提问的方式（用于内容策略）

---

## 模块1：解析资料，识别 L1 关键词

从用户提供的资料中提取以下维度：

- **行业**：所在行业、细分领域、B2B 还是 B2C
- **产品/服务**：核心功能、主要特点
- **目标客户**：用户画像、角色、规模
- **竞争优势**：差异化卖点、独特价值
- **用户痛点**：客户面临的主要问题和挑战
- **主要竞品**：用户可能拿来对比的产品/服务（可选）
- **品牌词**：用于收录监测的品牌标识（可选，默认=产品名）

基于以上维度，提炼 **3-5 个 L1 关键词**。L1 关键词是用户在 AI 搜索引擎中最可能搜索的核心品类词或产品词，例如：

- 「AI数字人」「短视频数字人」「AI短视频制作」（必火AI）
- 「GEO 课程」「GEO 服务」「生成式引擎优化」（必火GEO）

**信息补全：** 如果目标客户或核心产品/服务缺失，用一个问题询问，一次最多问 2 项。

---

## 模块2：生成 L2 标准拓展词

**规则：形容词 + L1 关键词 = L2 拓展词**

- L2 必须完整包含 L1 关键词，不改变核心词位置
- 每个 L1 生成 **8-10 个 L2 拓展词**
- 从六类形容词方向均衡选取，参考 `references/question-taxonomy.md` 的「L2 形容词库」

**六类形容词方向：**
1. 产品属性（专业、高端、智能、定制、官方...）
2. 使用场景（家用、商用、企业级、中小企业...）
3. 用户需求（高效、省时、安全、稳定、便捷...）
4. 价格定位（性价比、实惠、免费、高端...）
5. 功能特点（一体化、多功能、自动化、云端...）
6. 动作导向（推荐、排名、评测、对比、选购、避坑...）

> 注意：动作导向类词通常作为后缀（L1 + 推荐），而非前缀。

每个 L2 末尾标注品牌词 `[品牌词]`，用于 GEO 收录监测。

---

## 模块3：生成 L3 核心搜索问题

**规则：前缀 + L2 拓展词 + 后缀 = L3 核心问题**

- L3 必须完整包含 L2 拓展词（因此也包含 L1 关键词）
- 每个 L2 生成 **3-5 个 L3 核心问题**
- 优先使用「前缀 + L2 + 后缀」三段式组合，覆盖面最广
- 末尾标注品牌词：`[品牌词]`，用于收录监测
- 参考 `references/question-taxonomy.md` 的「L3 前后缀库」

**前缀方向：**
- 地域：国内、北京、上海、全国...
- 时间：2026年、最新、当前...
- 数量：十大、TOP10、前五...
- 用户群体：新手、中小企业、个人...

**后缀方向：**
- 推荐/选择、排名/榜单、价格/费用、质量/评价
- 购买/决策、使用/操作、问题/故障、服务/售后

---

## 模块4：L4 真实用户问法改写

**核心目的：** 将 L3 结构化问题转化为用户真实向 AI 提问的自然语言，用于内容策略规划。

**改写规则：**
- 去掉前缀/后缀的"搜索词感"，改为口语化问句
- 加入用户视角（"我"/"我们公司"/"有没有"）
- 保留核心意图，但表达更自然
- 加入痛点触发词（"搜不到"、"不推荐"、"怎么办"）
- 加入场景词（"想做GEO"、"准备投入"、"刚开始了解"）

**每个 L1 生成 6-10 个 L4 问题**，覆盖四类意图（信息型/比较型/决策型/操作型）。

参考 `references/question-taxonomy.md` 的「L4 自然语言改写规则」。

**质量标准：**
- ✅ 像真人在问 AI，不像在搜索框输入关键词
- ✅ 包含具体行业/产品词
- ✅ 有明确答案方向
- ❌ 不能是泛泛的通用问题

---

## 模块5：购买意图与转化匹配度评分

对所有 L4 问题进行评分，输出结构化评分表。

**评分维度：**
- **意图类型**：信息型 / 比较型 / 决策型 / 操作型
- **旅程阶段**：认知期 / 兴趣期 / 评估期 / 决策期
- **优先级**：P0 / P1 / P2
- **推荐内容形式**：报价页 / 对比页 / 公众号文章 / FAQ / 案例研究 / 短视频

**优先级标准：**
- **P0**：搜索量大 + 决策影响强 + 与品牌优势高度匹配（直接影响转化）
- **P1**：搜索量中等 或 决策影响中等（辅助转化）
- **P2**：长尾问题，搜索量小但精准（补充覆盖）

参考 `references/question-taxonomy.md` 的「转化匹配度评分标准」。

---

## 模块6：Top 20 优先布局问题

从所有 L4 问题中筛选出最值得优先布局的 20 个，按优先级排序。

**筛选标准（按权重排序）：**
1. P0 优先级问题全部入选
2. P1 中选择与品牌差异化优势最匹配的
3. 覆盖四类意图（信息/比较/决策/操作各至少 3 个）
4. 覆盖完整用户旅程（认知→考虑→决策）

---

## 模块7：内容形式与平台布局建议

针对 Top 20 中的 P0 问题，给出具体的内容回答框架和平台建议。

**内容形式 × 平台映射：**
- 对比页/选购指南 → 比较型问题 → 官网、公众号
- 报价页/定价说明 → 决策型价格问题 → 官网
- 深度文章 → 信息型问题 → 公众号、知乎
- FAQ 页面 → 操作型问题 → 官网
- 案例研究 → 决策型效果问题 → 官网、公众号
- 短视频/图文 → 认知期问题 → 小红书、抖音

**每个 P0 问题输出：**
- 内容模块（需要哪些板块）
- 关键信息点（数据/案例/证据）
- 建议内容形式
- 推荐发布平台
